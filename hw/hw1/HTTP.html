Hyper Text Transfer Protocol

Hyper Text Transfer Protocol is a standard that web browsers and web servers use to communicate. It is used by the client program to request files from a remote server. HTTP is a stateless system meaning that unlike some other protocols, the HTTP request is dropped once the request is completed.

How HTTP works

Clients and servers communicate using request and respond messages. The three main messages are GET, POST, and head.

  • HTTP GET messages sent to a server contain only a URL. Optional parameters may be added to the end of the URL. The server takes the optional data portion of the URL, if present, and returns the result to the browser.
  • HTTP POST messages place any parameters in the body of the request message rather than adding them to the end of the URL.
  • HTTP HEAD request works the same as GET requests. Instead of replying with the full contents of the URL, the server sends back only the header information.
